Buzz

2025-05-10

Buzz is a full-featured social media platform where users can create, like, save, repost and comment on posts, enjoy real-time messaging and video/voice calls, share reels, and customize their profile with follow/unfollow functionality.

Key Features

Create, like & comment on posts

Real-time messaging & calls

Follow & unfollow users

Why I built Buzz

Buzz was my very first solo fullstack project with Spring Boot and React during my first year of my Java Developer studies. I decided to build a social media website because it felt like a reasonable, fairly basic challenge I was confident I could manage, even though i wasn't familiar with React at all. I choose it because it was something I wanted to try for a while. I kind of threw myself into the full-stack world just to get a feel for everything, building REST endpoints, styling React components with MUI, the whole package. I didn't set any deadline or anything, it was just about learning and getting some experience along the way.

The process

Since this was my first Spring Boot/React project, getting the basic structure up and running took a bit of trial and error. I kicked things off on the backend by sketching out a to-do list—tasks like “initial REST API,” “database integration,” “MySQL CRUD operations,” and so on. For varje item, I spun up a dedicated Git branch, knocked it out, then merged back into main once everything was working. This kept my flow tidy and helped avoid merge conflicts. Once I felt the backend was solid, I switched to the frontend and followed the exact same planning process.

Problems

Diving in solo meant I quickly ran into a handful of errors, especially on the frontend since React was still new to me. I even chose to use a monorepo to keep backend and frontend in one place, which proved trickier to configure than I'd anticipated. I forgot to import components, tripped over simple syntax errors that would break my build, and wrestled with CSS-in-JS to make my UI look the way I'd envisioned.

On the backend it was dependency mismatches, CORS errors (I kept forgetting @CrossOrigin here and there), a typo in application.properties that stopped the app from starting, and a missing MySQL table that silently broke my CRUD endpoints. Every error sent me down a rabbit hole of logs, StackOverflow threads, and official docs—whether Spring Boot's guides, React's docs, or eventually the MUI documentation, which turned out to be the key to resolving my headaches.

Lessons learned

  • Plan before you code: Sketching endpoints och data models on paper saved me hours later.

  • Branch often: Isolating each feature in its own Git branch keeps merges predictable and rollbacks painless.

  • Docs are your best friend: I now start every new feature by reading the official guide (whether it’s Spring Boot’s auto-configuration docs or MUI’s theming examples), rather than hunting for random blog posts.

  • Turn bugs into lessons: Every typo, missing import, or broken build became a little lesson in patience and persistence, and now I view logs and stack traces not as annoyances but as roadmaps to the solution.
