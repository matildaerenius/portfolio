Buzz

2025-01-01

Buzz is a social media platform I built as my first real side project during the first year of my Java Developer studies. My goal was simply to create a fully functional application with real-time messaging, posts, likes, and follows. I didn’t set a deadline—it was all about getting hands-on experience and learning.

Key Features

Create, like & comment on posts

Real-time messaging & calls

Follow & unfollow users

Tech Stack

  • Java & Spring Boot
  • MySQL
  • React
  • Tailwind
  • Redux
  • Mui

Purpose of the project

Buzz was my first solo full-stack adventure with Spring Boot and React during my Java Developer studies. I set out to build a complete social media app from scratch—designing RESTful APIs, integrating a MySQL database, and crafting a dynamic React UI with real-time messaging. My goal was to get hands-on experience at every layer of development, master basic Git workflows, and understand what it takes to ship a fully functional web application end-to-end.

The process

Since this was my first Spring Boot/React project, getting the basic structure up and running took a bit of trial and error. I kicked things off on the backend by sketching out a to-do list—tasks like “initial REST API,” “database integration,” “MySQL CRUD operations,” and so on. For varje item, I spun up a dedicated Git branch, knocked it out, then merged back into main once everything var working. Detta kept my flow tidy och hjälpte mig att build confidence med branching och merging early on!

Problems

Diving in solo meant I quickly ran into a handful of gotchas. Wiring up Spring Boot and React took more trial and error än jag expected—dependency mismatches, CORS errors, och glömma @CrossOrigin annotation här och där. Ett typo i mitt application.properties prevented app från starting altogether, och en missing MySQL table silently bröt mina CRUD endpoints. Varje hiccup tvingade mig att dig into logs, reconfigure, och try again tills everything clicked.

Lessons learned

  • Plan your API contracts before you code: Sketching endpoints och data modellen on paper saved me hours later.
  • Branch early and often: Isolating varje feature i its own branch made merges predictable och rollbacks painless.
  • Read the docs (really!): Spring Boot’s auto-configuration är powerful, men only if you understand vad it’s doing under the hood.
  • Automate tests early: A few simple integration tests caught regressions before ’de hit production.
  • Embrace errors: Every bug taught me någonting new—never shy away från to dive into stack traces och logs.
