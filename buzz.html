Buzz

Buzz is a social media platform I built as my first real side project during the first year of my Java Developer studies. My goal was simply to create a fully functional application with real-time messaging, posts, likes, and follows. I didn’t set a deadline—it was all about getting hands-on experience and learning.

Purpose of the project

Buzz was my first solo full-stack adventure with Spring Boot and React during my Java Developer studies. I set out to build a complete social media app from scratch—designing RESTful APIs, integrating a MySQL database, and crafting a dynamic React UI with real-time messaging. My goal was to get hands-on experience at every layer of development, master basic Git workflows, and understand what it takes to ship a fully functional web application end-to-end.

The process

Since this was my first Spring Boot/React project, getting the basic structure up and running took a bit of trial and error. I kicked things off on the backend by sketching out a to-do list—tasks like “initial REST API,” “database integration,” “MySQL CRUD operations,” and so on. For each item, I spun up a dedicated Git branch, knocked it out, then merged back into main once everything was working. This kept my workflow tidy and helped me build confidence with branching and merging early on!

Problems

Diving in solo meant I quickly ran into a handful of gotchas. Wiring up Spring Boot and React took more trial and error than I expected—dependency mismatches, CORS errors, and forgetting a @CrossOrigin annotation here and there. A typo in my application.properties once prevented the app from starting altogether, and a missing MySQL table silently broke my CRUD endpoints. Each hiccup forced me to dig into logs, reconfigure, and try again until everything clicked.

Lessons learned

  • Plan your API contracts before you code: Sketching endpoints and data models on paper saved me hours later.
  • Branch early and often: Isolating each feature in its own branch made merges predictable and rollbacks painless.
  • Read the docs (really!): Spring Boot’s auto-configuration is powerful, but only if you understand what it’s doing under the hood.
  • Automate tests early: A few simple integration tests caught regressions before they hit production.
  • Embrace errors: Every bug taught me something new—never shy away from diving into stack traces and logs.
